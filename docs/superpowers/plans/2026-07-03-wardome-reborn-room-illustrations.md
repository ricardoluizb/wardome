# Wardome Reborn — Room Illustrations Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Show a real, AI-generated dark-fantasy illustration of the player's current room in the browser client's side panel, for the walkable Temple of Midgaard → Newbie Zone path, with a graceful fallback image for every other room in the game.

**Architecture:** A standalone, one-shot Node script (`tools/gen-room-art.js`) reads the real extracted room data (`extract/out/rooms/<vnum>.json`) for the MVP room slate, builds a dark-fantasy image prompt from each room's actual name and description, and fetches a generated JPEG from Pollinations.ai's free, key-less, no-signup image API (`https://image.pollinations.ai/prompt/<prompt>`). The resulting JPEGs are committed as static assets under `web/assets/rooms/<vnum>.jpg`, plus one `placeholder.jpg` for rooms outside the MVP slate. The existing browser client (`web/client.js`), which already receives `{"type":"room","id":<vnum>}` messages from the bridge (built in the foundation plan), is extended to swap an `<img>` element's `src` to the matching asset — or the placeholder — every time the room changes. No new runtime dependency, no build step, no server-side image generation: the art is pre-generated once and served as static files, same as any other frontend asset.

**Tech Stack:** Node.js 25 (built-in `fetch`, no npm dependency) for the one-shot art-generation script; Pollinations.ai (free, unauthenticated HTTP image-generation API — confirmed reachable and producing usable dark-fantasy-style JPEGs during planning); plain HTML/CSS/JS (existing `web/` client, no framework).

## Global Constraints

- No changes to `wdii/src/*` in this plan — this is a pure frontend/tooling addition, the room-ID tag already exists from the foundation plan.
- `web/` stays dependency-free (no npm install, no bundler) — the art script lives in `tools/` and is run manually/once, it is not part of the browser bundle.
- The MVP room slate is exactly these 9 vnums, confirmed via a live walk of `extract/out/rooms/*.json` exit graphs during planning: `3001` (Temple Of Midgaard), `3054` (By The Temple Altar), `3059` (Behind The Temple Altar), `3060` (The Great Field Of Midgaard), `3061` (The Great Field Of Midgaard), `18600` (The Entrance To The Newbie Zone), `18601` (The Beginning Of The Passage), `18602` (The Dirty Hallway), `18603` (A Nexus). Note: this is fewer than the ~16 rooms estimated in earlier planning notes — several rooms deeper in zone 186 (e.g. `18605`, `18606`, `18607`) have no corresponding file under `extract/out/rooms/` (a gap in the original extraction, likely from door-command parsing quirks in the `.wld` source), so they are not usable as real extracted content and are excluded rather than invented.
- Every room in the game that is *not* in the MVP slate must show a graceful placeholder image, never a broken image icon — the bridge tags every room the player enters, not just the MVP ones.
- Pollinations.ai requires no API key and has no documented cost — do not introduce a paid or key-gated image provider without asking first.
- Never commit unless explicitly instructed.

- [ ] **Step 1: Write the generation script**

```javascript
// tools/gen-room-art.js
const fs = require('fs');
const path = require('path');

const ROOM_IDS = [3001, 3054, 3059, 3060, 3061, 18600, 18601, 18602, 18603];
const OUT_DIR = path.join(__dirname, '..', 'web', 'assets', 'rooms');
const ROOMS_DIR = path.join(__dirname, '..', 'extract', 'out', 'rooms');

const STYLE_SUFFIX =
  "dark fantasy RPG environment concept art, digital painting in the style of " +
  "Baldur's Gate 3 and Disco Elysium, moody atmospheric lighting, painterly detail, " +
  "wide shot, no text, no UI, no people, no characters";

async function fetchImage(prompt, seed, outPath) {
  const url = `https://image.pollinations.ai/prompt/${encodeURIComponent(prompt)}?width=768&height=512&seed=${seed}&nologo=true`;
  const res = await fetch(url);
  if (!res.ok) {
    throw new Error(`HTTP ${res.status} fetching seed ${seed}`);
  }
  const buf = Buffer.from(await res.arrayBuffer());
  fs.writeFileSync(outPath, buf);
  console.log(`[gen-room-art] wrote ${outPath} (${buf.length} bytes)`);
}

async function main() {
  fs.mkdirSync(OUT_DIR, { recursive: true });

  for (const id of ROOM_IDS) {
    const room = JSON.parse(fs.readFileSync(path.join(ROOMS_DIR, `${id}.json`), 'utf8'));
    const desc = room.description.replace(/\s+/g, ' ').trim();
    const prompt = `${room.name}. ${desc} ${STYLE_SUFFIX}`;
    await fetchImage(prompt, id, path.join(OUT_DIR, `${id}.jpg`));
    await new Promise((resolve) => setTimeout(resolve, 1000));
  }

  const placeholderPrompt = `A foggy, dim, unmapped stone corridor fading into darkness. ${STYLE_SUFFIX}`;
  await fetchImage(placeholderPrompt, 0, path.join(OUT_DIR, 'placeholder.jpg'));

  console.log('[gen-room-art] done');
}

main().catch((err) => {
  console.error('[gen-room-art] failed:', err.message);
  process.exit(1);
});
```
